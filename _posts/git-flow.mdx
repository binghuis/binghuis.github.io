Gitflow 是一种软件开发工作流程，它是基于 Git 版本控制系统的一种分支管理策略。Gitflow 定义了一套严格的分支模型，用于在项目中管理代码的不同版本和功能。它是 [Vincent Driessen](https://nvie.com/about/) 2010 年在文章 [A successful Git branching model](https://nvie.com/posts/a-successful-git-branching-model/) 中提出的。
