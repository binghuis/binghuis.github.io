从 0 到 1 实现一个迷你版 React

2023-05-23
本文整理于 Rodrigo Pombo 的 Build your own React

我们将从头开始，按照真实的 React 代码架构，逐步实现 React（不包括优化和非必要功能）。

不同于我之前的 “从头构建你自己的 React” 系列文章，本文基于 React 16.8，因此我们可以使用 Hooks ，并抛弃与类相关的所有代码。

你可以在 Didact 代码仓库中找到旧博客文章和代码的历史提交。这里还有一个关于相同内容的 演讲。

但这篇文章，是一个独立的内容。

从 0 开始，以下是我们将逐步实现的所有内容：

  • STEP 1：createElement 函数
  • STEP 2：render 函数
  • STEP 3：并发模式
  • STEP 4：Fibers
  • STEP 5：渲染和提交
  • STEP 6：协调过程
  • STEP 7：函数组件
  • STEP 8：Hooks

STEP 0: Review

首先让我们回顾一些基本概念。如果您已经对 React、JSX 和 DOM 元素的工作原理有很好的理解，可以跳过此步。

右边是一个 React 应用程序，只有三行代码。第一行定义了一个 React 元素。下一行从 DOM 中获取一个节点。最后一行将 React 元素渲染到容器中。

让我们删除所有 React 代码，并用原生 JavaScript 替换它。

_10

在第一行，我们有一个使用 JSX 定义的元素。它甚至不是有效的 JavaScript，首先我们需要将它替换为有效的 JavaScript 代码。

JSX 会被构建工具（如 Babel）转换为 JavaScript。这个转换通常很简单：将标签内的代码替换为对 createElement 的调用，传递 tagName、props 和 children 作为参数。

React.createElement 根据传入的参数创建一个对象。除了一些验证外，它不做其他操作。因此，我们可以安全地将函数调用替换为它的输出结果。

而一个元素就是一个对象，它有两个属性：type 和 props（实际上它还有更多属性，但我们只关心这两个）。

type 是一个字符串，指定我们想要创建的 DOM 节点的类型，就像你在使用 document.createElement 创建 HTML 元素时传递的标签名一样（它也可以是一个函数，我们将在第 7 步中介绍）。

props 是另一个对象，它包含了 JSX 属性中的所有键值对。它还有一个特殊的属性：children。

本例中 children 是一个字符串，但通常它是一个包含更多元素的数组。这就是为什么元素也是树状结构的原因。
