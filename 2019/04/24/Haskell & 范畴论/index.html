<翻译> ：Haskell / 范畴论

Posted on 2019-04-24

原文：Haskell/Category_theory
本文将概述 Haskell 里应用的一个概念，范畴论。为此，Haskell 代码及其数学定义将会被一起呈现（不追求绝对的严谨）。我们仅试图让读者直观地了解范畴论的概念以及它与 Haskell 的关系。

范畴论是什么


上图为一个简单的范畴，包含三个对象 A，B，C，三个单位态射（identity morphisms）$id_A$, $id_B$和$id_C$，还有两个其他的态射$f : C \to B$和$g : A \to B$，图中没有展示第三个组成元素（即如何对态射进行组合）。

范畴，本质上是一个简单的集合，包括三个组成元素：

  • 对象的集合
  • 态射的集合，每个态射将两个对象（源对象和目标对象）连接在一起（这些有时被称为箭头（arrows），但我们在这里避免使用该术语，因为它在 Haskell 中具有其他内涵。）如果 $f$ 是源对象 $A$ 到目标对象 $B$ 的态射，写作 $f : A \to B$。
  • 态射组合的概念。例如：态射 $g : A \to B$ 和 态射 $f : B \to C$ 可以组合为态射 $f \circ g : A \to C$。范畴由许多东西构成。例如，全部的集合构成了范畴 Set

范畴规则

范畴需要符合三条规则。第一条，也是最简单的一条，态射的组合操作需要满足结合律。

$f\circ (g\circ h)=(f\circ g)\circ h$

态射在 Haskell 中从右到左执行（最常用于数学），因此使用$f\circ g$时，$g$先执行，然后$f$。

第二条，态射在组合操作下是闭合的。因此，如果存在态射$f:B \to C$和$g:A \to B$，那么范畴$h = f \circ g$中一定会存在态射$h : A \to C$。以下面范畴为例。


$f$和$g$都是态射，所以我们一定能够通过组合他们在范畴中得到另一个态射。
那么哪一个是态射$f \circ g$呢？唯一的选项就是$id_A$。同样，我们可以得到$g \circ f = id_B$。

最后一条，在一个范畴 C 中，每一个对象 A 都会有一个身份态射，$id_A : A \to A$，这个态射是组合操作的单位元。准确的说对于每一个态射$g: A \to B $：

$g \circ id_A = id_B \circ g = g$

请注意，涉及组合操作的表达式$（o）$可以彼此相等，但各个态射不能相互相等。例如有两个态射从对象$A$到对象$B$，即$f : A \to B$ 和 $g : A \to B$，表达式$(A \to B)$相同，但态射$f = g$永远为假。
